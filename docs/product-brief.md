# Product Brief

## Vision

`frontend-lab`은 여러 프론트엔드 프로젝트에서 반복되는 UI 구현을 줄이고, 일관된 사용자 경험을 제공하기 위한 디자인 시스템 실험실입니다.

현재 목표는 완성된 패키지 배포보다 컴포넌트 구조, 문서 계약, 디자인 토큰, 접근성 기준을 먼저 안정화하는 것입니다.

## Primary Users

- 제품 화면을 구현하는 프론트엔드 개발자
- 개인 프로젝트에서 빠르게 UI 기반을 잡고 싶은 개발자
- 컴포넌트 상태와 사용 기준을 검토하는 디자이너
- 화면 구성과 UI 일관성을 확인하는 기획자 또는 PM

## Design Principles

### Predictable

컴포넌트 이름, prop, 상태, 이벤트는 예측 가능해야 합니다. 같은 의미의 동작은 같은 이름과 패턴을 사용합니다.

### Accessible by Default

키보드 조작, 포커스 표시, 스크린 리더 의미, 오류 상태를 기본 설계에 포함합니다. 접근성은 별도 옵션이 아니라 컴포넌트의 기본 품질 기준입니다.

### Token Driven

컴포넌트는 원시 스타일 값을 직접 소유하지 않습니다. 색상, 간격, radius, shadow, motion은 `packages/tokens`의 `--ds-*` 토큰을 기준으로 사용합니다.

### Composable

작은 컴포넌트를 조합해 복잡한 UI를 만들 수 있어야 합니다. 컴포넌트는 특정 서비스 로직이나 데이터 구조에 과하게 묶이지 않아야 합니다.

### Documented

사용법, 상태, 제한 사항, 접근성 요구사항을 컴포넌트 문서에 남깁니다. 설계 결정은 문서로 기록해 나중에 맥락을 복구할 수 있게 합니다.

## Success Criteria

- 신규 화면에서 기본 입력, 버튼, 선택, 피드백 UI를 직접 새로 만들지 않아도 됩니다.
- 컴포넌트별 문서만 읽어도 목적, prop, 상태, 접근성 요구사항을 이해할 수 있습니다.
- 토큰 변경이 컴포넌트 스타일과 문서 예제에 일관되게 반영됩니다.
- 주요 컴포넌트는 접근성 체크리스트를 통과합니다.
- 로컬 문서 앱에서 로드맵, 규약, 예시를 빠르게 확인할 수 있습니다.
