# 언어 정책 / Language Policy

이 저장소의 기본 언어는 한글이며, 협업과 재사용을 위해 영문을 함께 제공합니다.
The default language of this repository is Korean, with English provided alongside it for collaboration and reuse.

## 적용 범위 / Scope

- 문서: README, 설계 문서, 컴포넌트 스펙, ADR, 템플릿에 모두 적용합니다. / Documentation: Applies to README files, design docs, component specs, ADRs, and templates.
- 주석: 코드 주석, TODO, NOTE는 한글을 먼저 쓰고 영문을 이어 씁니다. / Comments: Code comments, TODOs, and NOTEs should place Korean first and English second.
- 개발자 메시지: CLI 출력, 검증 메시지, 문서 앱에서 개발자가 읽는 안내 문구에도 같은 기준을 적용합니다. / Developer messages: CLI output, validation messages, and developer-facing copy in the docs app should follow the same standard.
- 예외: 코드 식별자, API 이름, 패키지명, 토큰명, 표준 링크 제목은 원문을 유지할 수 있습니다. / Exceptions: Code identifiers, API names, package names, token names, and standard link titles may stay in their source language.

## 작성 형식 / Writing Format

- 제목은 `한글 제목 / English Title`로 씁니다. / Write headings as `Korean Title / English Title`.
- 본문은 한글 문장을 먼저 쓰고 같은 문단 또는 같은 bullet 뒤에 영문을 붙입니다. / Put the Korean sentence first, then add the English sentence in the same paragraph or bullet.
- 표의 열 제목은 가능하면 `한글 / English` 형식으로 병기합니다. / Use `Korean / English` column labels in tables when practical.
- 짧은 UI 라벨은 공간이 부족하지 않으면 `한글 / English`로 씁니다. / Use `Korean / English` for short UI labels when space allows.

## 새 작업 기준 / Future Work Standard

- 새 문서와 새 주석은 처음부터 병기 형식으로 작성합니다. / Write new docs and comments in the bilingual format from the start.
- 기존 문서를 수정할 때는 수정한 주변 문단도 같은 기준으로 맞춥니다. / When editing existing docs, align nearby edited paragraphs with the same standard.
- 컴포넌트 README와 spec은 스캐폴드 스크립트가 생성하는 병기 템플릿을 기준으로 합니다. / Component README and spec files should follow the bilingual templates generated by the scaffold script.
