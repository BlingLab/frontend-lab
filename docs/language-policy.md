# 언어 정책

이 저장소의 기본 언어는 한글입니다. 문서와 개발자-facing 메시지는 한글만 사용해 읽는 흐름을 짧게 유지합니다.

## 적용 범위

- 문서: README, 설계 문서, 컴포넌트 스펙, ADR, 템플릿에 모두 적용합니다.
- 주석: 코드 주석, TODO, NOTE는 한글로 작성합니다.
- 개발자 메시지: CLI 출력, 검증 메시지, 문서 앱에서 개발자가 읽는 안내 문구에도 같은 기준을 적용합니다.
- 예외: 코드 식별자, API 이름, 패키지명, prop 이름, CSS class, token 이름, URL, 표준 링크 제목은 원문을 유지할 수 있습니다.

## 작성 형식

- 제목은 `한글 제목`으로 씁니다.
- 본문은 한글 문장만 사용합니다.
- 표의 열 제목은 한글로 씁니다.
- 예시 UI 라벨도 한글로 씁니다.
- 공식 영문 용어가 필요한 경우에는 `accessible name`, `focus-visible`, `controlled`처럼 원문 용어를 유지하고 설명만 한글로 붙입니다.

## 새 작업 기준

- 새 문서와 새 주석은 처음부터 한글 전용으로 작성합니다.
- 기존 문서를 수정할 때는 수정한 주변 문단도 같은 기준으로 맞춥니다.
- 컴포넌트 README와 spec은 스캐폴드 스크립트가 생성하는 한글 전용 템플릿을 기준으로 합니다.
